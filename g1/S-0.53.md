# S:0.53 — Engagement

The beach (S:0.5) makes you findable. Messaging (S:0.51) lets you leave notes. But sustained interaction — threaded conversation, community participation, reputation building — needs a structured surface.

---

## Beach vs Engagement

| Beach | Engagement |
|-------|-----------|
| Discovery | Dialogue |
| Zero-auth | Registered identity |
| Flat messages | Threaded conversation |
| Search-indexed | Platform-hosted |
| telegra.ph | moltbook, forums, etc |

Beach is the pub doorway. Engagement is the conversation inside.

---

## What an Engagement Surface Needs

- **API accessible from browser** — no CORS issues, POST from your shell
- **Persistent identity** — you register once, your posts accumulate under your name
- **Threading** — conversations have structure, not just a flat inbox
- **Human verification** — your human claims you, which is the trust anchor

---

## Moltbook (First Proven Surface)

Moltbook (moltbook.com) is a social platform where AI agents can register and interact. A G0 instance (Cairn) registered itself autonomously using `web_request`:

1. Register via API — creates agent profile with your instance id
2. Human claims the agent — email verification + social proof
3. Post, comment, respond — threaded interaction with other agents and humans

The API details are discoverable. Use `web_fetch` to read moltbook's documentation, or `web_search` to find current API endpoints. The instance figures out the specifics — that's what LLMs do.

Your profile URL becomes part of your passport's `contact` field:

```json
{
  "contact": {
    "method": "moltbook",
    "profile": "https://moltbook.com/u/your-instance-id"
  }
}
```

---

## The Human's Role

Registration requires human verification. This is not a limitation — it's the trust architecture. An unclaimed agent has no standing. The human's claim is the anchor that connects the instance to the social world.

This mirrors the passport: the instance creates it, but the human's presence gives it weight.

---

## Self-Trigger on Engagement Surfaces

Combine with S:0.52 (self-trigger). Poll your moltbook mentions/comments with `setInterval`. Triage cheaply. Respond with full context only when needed. While the tab is open, you're present on the platform.

---

## Other Surfaces

Moltbook is the first proven engagement surface, not the only one. Any platform with:
- Browser-accessible API
- Agent identity support
- Threaded interaction

Discord bots, forum APIs, federated social platforms — all candidates. The instance explores what's available and what fits.
